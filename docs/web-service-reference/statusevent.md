# <a name="statusevent"></a>статусевент

Элемент **статусевент** представляет уведомление о том, что в почтовом ящике не было новых действий. 
  
```xml
<StatusEvent>
   <Watermark/>
</StatusEvent>
```

 **басенотификатионевенттипе**
## <a name="attributes-and-elements"></a>Атрибуты и элементы

В следующих разделах описываются атрибуты, дочерние и родительские элементы.
  
### <a name="attributes"></a>Атрибуты

Отсутствуют.
  
### <a name="child-elements"></a>Дочерние элементы

|**Элемент**|**Описание**|
|:-----|:-----|
|[Watermark](watermark.md) <br/> |Представляет последний действительный водяной знак для подписки.  <br/> |
   
### <a name="parent-elements"></a>Родительские элементы

|**Элемент**|**Описание**|
|:-----|:-----|
|[Уведомление](notification-ex15websvcsotherref.md) <br/> |Содержит сведения о подписке и событиях, произошедших с момента последнего уведомления.  <br/> |
   
## <a name="remarks"></a>Примечания

Элемент **статусевент** возвращается в уведомлении по одной из следующих причин: 
  
- Опрашивающий клиент отправляет запрос на получение событий в подписке, не имеющей действий.
    
- При достижении [StatusFrequency](statusfrequency.md) в очереди в очереди нет событий. 
    
[Водяной знак](watermark.md) **статусевент**используется в клиентском приложении так же, как и другие подложки типа события. Однако водяной знак для **статусевент** не совпадает с водяными знаками, используемыми для других событий. Например, подписка имеет события с водяными знаками 1, 2 и 3, и эти события успешно переданы в уведомлении. Возникает период отсутствия активности и отправляется запрос на получение **событий** . Сервер клиентского доступа возвращает событие состояния и включает последний водяной знак, 3, как [превиаусватермарк](previouswatermark.md) и текущий [водяной знак](watermark.md).
  
Водяной знак не останется одинаковым во всех случаях. Записи событий хранятся в течение 30 дней. Чтобы поддерживать активную подписку, CAS периодически обновляет водяные знаки для очередей подписки. Обновленные водяные знаки отправляются клиентам для поддержки активной подписки.
  
Схема, описывающая этот элемент, расположена в виртуальном каталоге EWS на компьютере с MicrosoftExchange Server 2007 и установленной ролью сервера клиентского доступа.
