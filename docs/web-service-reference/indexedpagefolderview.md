# <a name="indexedpagefolderview"></a>индекседпажефолдервиев

Элемент **индекседпажефолдервиев** описывает, как возвращаются сведения о страничном элементе в отклике [FindFolder](findfolder.md) . 
  
[FindFolder](findfolder.md)
  
[индекседпажефолдервиев](indexedpagefolderview.md)
  
```xml
<IndexedPageFolderView MaxEntriesReturned="" Offset="" BasePoint="" />
```

 **индекседпажевиевтипе**
## <a name="attributes-and-elements"></a>Атрибуты и элементы

В следующих разделах описываются атрибуты, дочерние и родительские элементы.
  
### <a name="attributes"></a>Атрибуты

|**Атрибут**|**Описание**|
|:-----|:-----|
|**максентриесретурнед** <br/> |Описывает максимальное число папок, возвращаемых в ответе. Этот атрибут является необязательным.  <br/> |
|**Offset** <br/> |Описывает смещение от **басепоинт**. Смещение должно быть больше или равно нулю. Если **басепоинт** равно "начало", смещение является положительным. Если **басепоинт** равен end, то смещение обрабатывается так, как если бы оно было отрицательным.  <br/> Этот параметр определяет, какая папка будет первой папкой, доставляемой в ответе. Этот атрибут является обязательным.  <br/> |
|**басепоинт** <br/> |Указывает, будет ли страница папок начинаться с начала или с конца набора папок, найденных с помощью условий поиска. Поиск от End всегда выполняет поиск в обратном направлении. Этот атрибут является обязательным.  <br/> |
   
#### <a name="basepoint-attribute"></a>Атрибут Басепоинт

|**Значение**|**Описание**|
|:-----|:-----|
|Слайд  <br/> |Постраничное представление начинается с начала найденного набора папок.  <br/> |
|Конец  <br/> |Постраничное представление начинается в конце найденного набора папок.  <br/> |
   
### <a name="child-elements"></a>Дочерние элементы

Нет.
  
### <a name="parent-elements"></a>Родительские элементы

|**Элемент**|**Описание**|
|:-----|:-----|
|[FindFolder](findfolder.md) <br/> |Определяет запрос на поиск папок в почтовом ящике.  <br/> Ниже приведено выражение XPath для этого элемента:  <br/>  `/FindFolder` <br/> |
   
## <a name="remarks"></a>Примечания

Поиск с конца включает переход к началу, определяемому смещением. Кроме того, указатель перемещается обратно на число запрошенных записей. Например, если имеется 100 записей, а смещение — 25 из конца, поиск начинается с 75. Если возвращено 10 записей, указатель перемещается обратно в обратное 10 записей в 65 и возвращает записи от 65 до 75. Следующий индекс — 64. Следующее смещение от конца для страницы составляет 100 минус 64, что равно 36. Значение для следующего смещения из конца для получения следующей индексированной страницы — 36.
  
Схема, описывающая этот элемент, расположена в виртуальном каталоге EWS на компьютере с MicrosoftExchange Server 2007 и установленной ролью сервера клиентского доступа.
