# <a name="persistent-application-settings-in-ews-in-exchange"></a>Параметры сохраняемого приложения в веб-службах Exchange

Узнайте о различных параметрах, управляемый API EWS или приложение веб-служб Exchange можно использовать для создания сохраняемых пользовательские параметры приложения в Exchange.
  

  
Для сохранения параметров приложения на сервере Exchange — это самый простой способ синхронизации конфигурации пользовательских клиентов для почтового ящика и папок и элементов в почтовом ящике. Можно убедиться, что эти параметры сохранения для почтового ящика, используя один из следующих: 
  
- Настройка пользовательских объектов
    
- Расширенные свойства
    
- Настраиваемые элементы
    
## <a name="what-are-my-options-for-creating-persistent-application-settings"></a>Что такое параметры для создания параметров сохраняемый приложения?
<a name="Options"> </a>

Объекты конфигурации пользователя, лучше для хранения параметров конфигурации для клиентских приложений, веб-служб Exchange. Вы также можете использовать расширение свойств или настраиваемые элементы и их комбинации все три. Выберите параметр в области Параметры и ли параметры должны быть доступны для других приложений.
  
**В таблице 1. Рекомендуемые параметры для создания параметров сохраняемый приложения на основе области действия**

|**Задание области**|**Используйте...**|**Доступ к**|
|:-----|:-----|:-----|
|Элемент  <br/> |Расширенное свойство в существующий элемент.  <br/> |Любое приложение веб-служб Exchange. Только клиентов веб-служб Exchange, которые знать идентификатор свойства можно получить доступ к расширенному свойству.  <br/> |
|Folder  <br/> |Объект конфигурации пользователей в целевой папке. Это удобный способ сохранения параметров представления для папки.  <br/> |Любое приложение веб-служб Exchange.  <br/> |
|Почтовый ящик  <br/> |Объект конфигурации пользователей в общей папке msgrootfolder по умолчанию.  <br/> |Любое приложение веб-служб Exchange.  <br/> |
   
### <a name="user-configuration-objects"></a>Настройка пользовательских объектов
<a name="UserConfig"> </a>

Объекты конфигурации пользователя, специальные элементы, связанные с папками в почтовом ящике. Конфигурация, объектов пользователей, также известной как элементы папки, связанной обычно — лучший вариант для сохранения параметров приложения, особенно если сведения о конфигурации связан с папкой или почтового ящика. Они обычно не отображаются для конечных пользователей. Так как они изначально может хранить потоки данных и данных словарей, они являются идеальным решением для хранения сведений о конфигурации. — Это лучший способ использования объектов конфигурации пользователя для хранения набора конфигураций в XML-документ, а затем сохраните эти сведения в одном из потока свойства конфигурации для пользователя.
  
Объекты конфигурации пользователей осуществляется иначе, чем другие типы элементов, хранятся в почтовом ящике. Метод управляемый API EWS [Folder.FindItems](http://msdn.microsoft.com/en-us/library/microsoft.exchange.webservices.data.exchangeservice.finditems%28v=EXCHG.80%29.aspx) или операции [FindItem](http://msdn.microsoft.com/library/ebad6aae-16e7-44de-ae63-a95b24539729%28Office.15%29.aspx) EWS можно использовать для поиска всех элементов, но необходимо использовать параметр обхода поиска **связанные** для объектов конфигурации пользователей. Обход поиска **связанные** указывает, что результаты поиска должны содержать только объектов конфигурации пользователей. Веб-служб Exchange включает набор операций, которые специфичны для объектов-пользователей конфигурации.

### <a name="extended-properties"></a>Расширенные свойства
<a name="ExtendedProperties"> </a>

Используйте [Расширенные свойства](properties-and-extended-properties-in-ews-in-exchange.md) , если необходимо для хранения сведений о конфигурации на элементы. Веб-служб Exchange, в отличие от MAPI, не возвращает контейнер свойств элементов. Это означает, что клиент веб-служб Exchange необходимо знать идентификатор расширенные свойства для поиска и доступа к расширенному свойству. Если вам требуется для хранения сведений о конфигурации для элементов, отличных от объектов конфигурации пользователя, с помощью расширенных свойств для создания настраиваемых свойств может быть решение для вас. Расширенные свойства позволяют получить доступ к и храните информацию на свойства, которые не являются частью стандартное свойство для элемента. 
  
> [!IMPORTANT]
> Схема базы данных Exchange имеет ограниченное число свойств. Свойство идентификаторов для базы данных Exchange не более 32 767. При использовании расширенных свойств для хранения много параметров, рекомендуется использовать один расширенные свойства для хранения этих параметров, чтобы не превышайте этом максимальное. 
  
Метод управляемый API EWS [Item.Update](http://msdn.microsoft.com/en-us/library/microsoft.exchange.webservices.data.item.update%28v=EXCHG.80%29.aspx) или операции EWS [UpdateItem](http://msdn.microsoft.com/library/5d027523-e0bc-4da2-b60b-0cb9fc1fdfe4%28Office.15%29.aspx) можно использовать для настройки расширенных свойств объектов конфигурации пользователей. 
  
### <a name="custom-items"></a>Настраиваемые элементы
<a name="CustomItems"> </a>

Настраиваемые элементы можно также использовать для хранения сведений. Существующие свойства элемента можно тегами для хранения сведений о конфигурации. Или можно использовать расширенные свойства для определения собственных свойств для приложения. Использование настраиваемых элементов для хранения конфигурации дает следующие преимущества: 
  
- Они работают для всех версий Exchange, которые поддерживают веб-служб Exchange.
    
- Если не использовать расширенные свойства для элемента, не прикреплен бюджета свойств Exchange Server.
    
## <a name="where-should-i-store-my-application-settings"></a>Где хранить параметры приложения
<a name="ApplicationSettingsLocation"> </a>

Папки почтовых ящиков и их элементов, находятся в корневой папке сообщения. Эта папка определяется значение [WellKnownFolderName.msgfolderroot](http://msdn.microsoft.com/en-us/library/office/microsoft.exchange.webservices.data.wellknownfoldername%28v=exchg.80%29.aspx) в управляемый API веб-служб Exchange. В терминах MAPI это эквивалентно поддерева IPM почтового ящика. Объекты конфигурации пользователей часто используются для создания пользовательского интерфейса на основе параметров, чтобы приложение может отобразить параметры представления на основании к папке, где пользователь получает доступ к. Параметры представления на основе папки обычно устанавливаются на объект конфигурации пользователя, который связан с папкой. Однако в некоторых случаях может потребоваться глобальные приложения выберите необходимые параметры. В этом случае можно хранить параметры в корневой папке сообщения. 
  
Большинство пользователей не поддерживают и обычно не доступа в корневой папке почтового ящика. Эта папка определяется значение [WellKnownFolderName.root](http://msdn.microsoft.com/en-us/library/office/microsoft.exchange.webservices.data.wellknownfoldername%28v=exchg.80%29.aspx) в управляемый API веб-служб Exchange. В терминах MAPI это эквивалентно поддерево не IPM почтового ящика. Сведения, которые конечные пользователи не прямой доступ к хранится в корневой папке почтового ящика. Можно хранить параметры приложения в этой папке, поскольку клиентских приложений не обычно получить к нему доступ. 
  
## <a name="version-differences"></a>Различия версий
<a name="VersionDifferences"> </a>

Объекты конфигурации пользователей доступны в Exchange Online, Exchange Online как часть Office 365 и версии Exchange, начиная с Exchange 2010.
