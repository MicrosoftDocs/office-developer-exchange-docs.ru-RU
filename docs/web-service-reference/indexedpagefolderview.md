# <a name="indexedpagefolderview"></a>IndexedPageFolderView

Элемент **IndexedPageFolderView** описывает, как выгружаемый элемента сведения возвращаются в ответ [FindFolder](findfolder.md) . 
  
[FindFolder](findfolder.md)
  
[IndexedPageFolderView](indexedpagefolderview.md)
  
```xml
<IndexedPageFolderView MaxEntriesReturned="" Offset="" BasePoint="" />
```

 **IndexedPageViewType**
## <a name="attributes-and-elements"></a>Атрибуты и элементы

В разделах ниже приводится описание атрибутов, дочерних и родительских элементов.
  
### <a name="attributes"></a>Атрибуты

|**Атрибут**|**Описание**|
|:-----|:-----|
|**MaxEntriesReturned** <br/> |Описывает максимальное число папок для возврата в ответе. Этот атрибут является необязательным.  <br/> |
|**Смещение** <br/> |Описание смещения от **Базисная точка**. Смещение должно быть больше или равно нулю. Если **Базисная точка** равна начала, смещение является положительным. Если **Базисная точка** равна End, смещение обрабатывается как если бы он был отрицательным.  <br/> Определяет, какие папки будут первой папки доставлено в ответе. Этот атрибут является обязательным.  <br/> |
|**Базисная точка** <br/> |Описывает ли страницы папок будет запущен процесс от начала или окончания в набор папок, найденных с условия поиска. При поиске с конца всегда ищет в обратном направлении. Этот атрибут является обязательным.  <br/> |
   
#### <a name="basepoint-attribute"></a>Атрибут Базисная точка

|**Значение**|**Описание**|
|:-----|:-----|
|Начало  <br/> |Постраничное представление начинается с начала набора обнаруженного папки.  <br/> |
|End  <br/> |Постраничное представление начинается в конце набора обнаруженного папок.  <br/> |
   
### <a name="child-elements"></a>Дочерние элементы

Нет.
  
### <a name="parent-elements"></a>Родительские элементы

|**Элемент**|**Описание**|
|:-----|:-----|
|[FindFolder](findfolder.md) <br/> |Определяет запрос для поиска папок в почтовом ящике.  <br/> Ниже приведен выражение XPath для этого элемента.  <br/>  `/FindFolder` <br/> |
   
## <a name="remarks"></a>Замечания

Поиск из плана включает в себя перемещение происхождения, определяемую средством смещение. Кроме того указатель возвращается назад на число запрошенных записей. Например если существует 100 записей и смещение равно 25 с конца, поиск начинается с 75. Если будет возвращено 10 записей, указатель мыши перемещается назад дополнительные 10 записывает 65 и возвращает записи 65 через 75. Следующий индекс — 64. Далее смещение с конца страницы равно 100 минус 64, которое равно 36. Значение для следующего смещения с конца для получения следующей страницы индексированных — 36.
  
Схема, описывающая этот элемент, расположена в виртуальном каталоге EWS на компьютере с MicrosoftExchange Server 2007 и установленной ролью сервера клиентского доступа.
