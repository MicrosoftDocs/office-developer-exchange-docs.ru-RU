# <a name="action-pox"></a>Действие (POX)

Элемент **Action** приведены сведения, которые используются для определения того, требуется ли другой запрос службы автообнаружения для возврата сведений о конфигурации пользователя. 
  
- [Автообнаружение (POX)](autodiscover-pox.md)
  
- [Ответ (POX)](response-pox.md)
  
- [Учетная запись (POX)](account-pox.md)
  
- [Действие (POX)](action-pox.md)
  
```xml
<Action>redirectUrl or redirectAddr or settings</Action>
```

## <a name="attributes-and-elements"></a>Атрибуты и элементы

В разделах ниже приводится описание атрибутов, дочерних и родительских элементов.
  
### <a name="attributes"></a>Атрибуты

Нет.
  
### <a name="child-elements"></a>Дочерние элементы

Нет.
  
### <a name="parent-elements"></a>Родительские элементы

|**Элемент**|**Описание**|
|:-----|:-----|
|[Учетная запись (POX)](account-pox.md) <br/> |Задает параметры учетной записи пользователя.  <br/> |
   
## <a name="text-value"></a>Текстовое значение

Текстовое значение представляет ли другой запрос автообнаружения, требуемой для получения сведений о конфигурации пользователя. В следующей таблице приведены возможные значения.
  
|**Значение**|**Описание**|
|:-----|:-----|
|redirectUrl  <br/> |Если указано это значение элемент [RedirectUrl (POX)](redirecturl-pox.md) будет укажите URL-адрес сервера клиентского доступа для использования в последующих запросов автообнаружения. Клиентское приложение должны прекратить перенаправление после 10 перенаправлений.  <br/> |
|redirectAddr  <br/> |Если указано это значение, элемент [RedirectAddr (POX)](redirectaddr-pox.md) будет укажите адрес электронной почты, который следует использовать для последующего запроса службы автообнаружения.  <br/> |
|settings  <br/> |Если указано это значение ответа службы автообнаружения содержит параметры, которые используются для настройки учетной записи. Большинство параметров можно найти в элементе [Протокола (POX)](protocol-pox.md) .  <br/> |
